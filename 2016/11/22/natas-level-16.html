The source code shows that many characters can't be used, yet following the theme of the previous challenges, we can inject code...


Unfortunately, it seems that injecting code with base64 encoded strings won't work, I tried something simple like $(base64 -d <<< YQo=) and didn't work. Another way is using boolean based injection, just like our previous example! For that to work, we need to find a way to answer our question. One way is the fact that it shows data if substring exists, otherwise it's empty.

Question: Does /etc/natas_webpass/natas17 contain the letter a?
Input: doomed$(grep a /etc/natas_webpass/natas17)
Output if Yes: <empty>
Output if No: doomed

Let's get our filtered character set first:
